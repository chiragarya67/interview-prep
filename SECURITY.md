# Security Policy

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you find a security issue (e.g. exposed API keys, auth bypass, data leak), report it privately:

1. Go to the **Security** tab of this repo
2. Click **"Report a vulnerability"**
3. Describe what you found

We'll acknowledge within 48 hours and work on a fix.

---

## What counts as a security issue?

- Authentication bypass
- Exposed environment variables or secrets
- API endpoints returning data they shouldn't
- XSS or injection vulnerabilities

## What doesn't need private reporting?

- UI bugs
- Broken features
- Performance issues

These can be reported as normal GitHub issues.
